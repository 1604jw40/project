# Chat Handoff

백엔드 공부 프로젝트를 다음 채팅에서 이어가기 위한 인계 문서다.

## 프로젝트 정보

프로젝트 루트:

```text
C:\backend_project
```

목표:

Express + TypeScript + PostgreSQL + Prisma + JWT 기반의 백엔드 프로젝트를 완성한다.

## 진행 방식

- 전체 챕터를 한 번에 보여주지 않는다.
- 한 번에 하나의 작업만 제시한다.
- 사용자가 실행 결과를 보내면 다음 작업으로 넘어간다.
- Python 기본기와 겹치는 설명은 줄인다.
- JavaScript와 백엔드에서 중요한 차이 위주로 설명한다.
- VS Code에서 파일을 직접 수정하므로 `notepad` 명령어는 사용하지 않는다.
- 필요한 디렉토리만 단계적으로 생성한다.
- 챕터가 끝나면 README.md와 이 문서를 갱신한다.
- 챕터가 끝나면 다음 채팅에서 바로 이어갈 수 있도록 이 문서를 갱신한다.
- 이 문서에는 완료한 챕터, 현재 학습 위치, 다음 챕터, 다음 채팅 시작 문구를 항상 최신 상태로 유지한다.
- 챕터 종료 후 커밋 메시지를 추천한다.
- 챕터 하나가 끝나면 다음 채팅으로 넘어간다.

## 현재 학습 위치

완료한 주요 내용:

- `let`과 `const` 차이

- 블록 스코프
- `var`를 새 코드에서 사용하지 않는 이유
- 조건에 따라 바뀌는 값은 `let` 사용
- 바뀌지 않는 값은 `const` 사용
- 백엔드 응답 형태를 객체로 구성하기
- 객체 속성 축약 문법
